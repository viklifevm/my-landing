## 1. Цель и аудитория

### Цель сайта

Конвертировать владельца малого или среднего бизнеса из "зашёл посмотреть" в "оставил заявку на консультацию" за одно посещение.

Вторичная цель — закрепить в голове клиента образ: **"это не просто разработчик, это человек с опытом в бизнесе, который знает, где теряются деньги"**.

### Целевая аудитория

**Первичная — владелец МСБ, который хочет автоматизировать бизнес:**
- Возраст: 30–55 лет
- Оборот компании: от 5 млн ₽/год
- Боли: менеджеры перегружены рутиной, клиенты долго ждут ответа, сайт не продаёт, нет понимания что автоматизировать
- Что ищет: не технологии, а результат — сэкономленные деньги и время
- Способ принятия решений: смотрит на примеры, читает отзывы, проверяет опыт
- Как связывается: Telegram, иногда Макс

**Вторичная — технический директор / руководитель IT-отдела:**
- Ищет внешнего подрядчика под конкретную задачу
- Важны: стек технологий, GitHub, конкретные кейсы с техническими деталями

### Что НЕ является аудиторией

- Студенты и стартапы без бюджета
- Крупный корпоративный бизнес с тендерами
- Другие разработчики и дизайнеры

## 2. Позиционирование

### Уникальная точка зрения

> Я строила бизнесы и знаю, где они теряют деньги.  
> Теперь я автоматизирую именно эти места с помощью ИИ.

Это не просто разработка. Это взгляд изнутри: экс-CEO, которая прошла путь предпринимателя, видела реальные потери — и теперь устраняет их цифровыми инструментами.

### Главный оффер (H1)

```
Я знаю, где ваш бизнес теряет деньги.
И знаю, как это починить с помощью ИИ.
```

### Подзаголовок (H2)

```
Создаю сайты, приложения и ИИ-агентов для малого и среднего бизнеса.
Под ключ. С опытом CEO — без лишней воды.
```

### Альтернативные варианты H1 для A/B-теста

- *"Бывший CEO. Теперь автоматизирую то, что раньше стоило вам денег."*
- *"80% рутины вашего менеджера можно убрать за 3 недели. Я знаю как."*
- *"Сайты, приложения и ИИ-агенты — от человека, который сам строил бизнес."*

### Ключевые отличия от конкурентов

| Обычный разработчик | Я |
|---|---|
| Говорит технологиями | Говорит результатами бизнеса |
| "Сделаю React-приложение" | "Сокращу время обработки заявок на 70%" |
| Видит задачу как код | Видит задачу как бизнес-процесс |
| Нет понимания бизнес-контекста | Опыт CEO: знаю, где болит |

### 00. Navigation

**Состав:**
- Логотип / имя (левый край)
- Ссылки: Обо мне · Услуги · Кейсы · Цены · Контакт
- Кнопка: "Обсудить проект" (правый край, акцентный цвет)

**Поведение:**
- Sticky при скролле
- При скролле вниз — фон появляется (glassmorphism: blur + полупрозрачный тёмный)
- Активный пункт меню подсвечивается при попадании секции в viewport
- На мобильном — бургер-меню с плавным раскрытием

### 01. Hero

**Задача:** За 3 секунды ответить: "Кто это, чем может помочь мне, и что делать дальше."

**Макет:** Две колонки на desktop (текст слева, фото справа), одна колонка на mobile (фото → текст).

**Контент:**

```
[Надпись над заголовком — мелкий текст с акцентом]
Экс-CEO · Разработчик · ИИ-интегратор

[H1 — крупно, доминирует]
Я знаю, где ваш бизнес
теряет деньги.

[H2 — чуть меньше, серый]
Создаю сайты, приложения и ИИ-агентов,
которые автоматизируют именно эти места.

[Строка доверия — мелко]
Для малого и среднего бизнеса · Под ключ

[CTA-блок]
[Кнопка PRIMARY] Обсудить проект →
[Кнопка SECONDARY] Посмотреть кейсы ↓

[Статистика — 3 числа]
7+ проектов    |    3 года в разработке    |    100% — бывший CEO
```

**Визуал:**
- Правая колонка: профессиональное фото автора (живое, не стоковое)
- Фон: тёмный с анимированными частицами-нейронами (Canvas, очень тонко — не отвлекает)
- На фото — лёгкое свечение по контуру (акцентный цвет)

**Анимация при загрузке:**
- Надпись над заголовком → fade in + slide up (0.3s)
- H1 — посимвольное появление (stagger, 0.6s)
- H2 — fade in (0.9s)
- Кнопки — fade in + scale from 0.95 (1.1s)
- Статистика — счётчики от 0 до значения (1.2s)

### 06. Process

**Задача:** Устранить страх "а вдруг это будет долго, непонятно и дорого".

**Макет:** Горизонтальная шкала с 5 шагами на desktop, вертикальная на mobile.

**Контент:**

```
[H2]
Как мы работаем

[Шаг 1]  🔍 Discovery — 1–3 дня
Созваниваемся, разбираем задачу. Я задаю вопросы как
бизнес-аналитик: где боль, что уже пробовали, какой
результат нужен. Без технического жаргона.
Вы получаете: чёткое понимание задачи и предварительную оценку

[Шаг 2]  📋 Предложение — 1–2 дня
Готовлю конкретное предложение: что делаем, как, за сколько,
в какие сроки. Прозрачная смета без скрытых доплат.
Вы получаете: документ с объёмом работ и стоимостью

[Шаг 3]  🛠 Разработка — 1–4 недели
Работаю итерациями. Каждую неделю — демо и синхронизация.
Вы всегда видите, что происходит. Изменения обсуждаем
сразу, не в конце.
Вы получаете: еженедельные обновления + доступ к черновику

[Шаг 4]  ✅ Запуск — 2–5 дней
Тестирование, правки, деплой. Передаю доступы и обучаю
команду работать с продуктом. Пишу инструкцию.
Вы получаете: готовый продукт + документация

[Шаг 5]  🔄 Поддержка — ongoing
Остаюсь на связи. Исправляю баги, помогаю развивать продукт.
Вы не остаётесь один на один с системой.
Вы получаете: ответ в течение 24 часов

Хотите узнать, как это будет выглядеть для вашей задачи?
[Кнопка] Получить бесплатную оценку →
```

## 5. Дизайн-система

### Настроение (Mood)

**Тёмное, технологичное, но человечное.** Не холодный корпоративный сайт. Не гиперактивный портфолио разработчика. Ощущение: "Серьёзный специалист, с которым приятно работать."

Ориентиры: Vercel + Linear + личный подход.

### Цветовая палитра

```
Основной фон:       #0a0a0f   (почти чёрный с синим подтоном)
Вторичный фон:      #111118   (карточки, секции)
Поверхность:        #1a1a24   (hover-состояния, выделения)
Граница:            #ffffff14 (тонкие разделители)

Акцент PRIMARY:     #a78bfa   (светло-фиолетовый — основные кнопки, ссылки)
Акцент SECONDARY:   #60a5fa   (голубой — теги, иконки, подсветка кода)
Акцент GLOW:        rgba(167, 139, 250, 0.15) (свечение вокруг элементов)

Текст PRIMARY:      #f8f8f8   (заголовки)
Текст SECONDARY:    #a0a0b0   (подписи, описания)
Текст MUTED:        #606070   (мета-информация)

Успех:              #34d399   (зелёный — метрики результатов)
Предупреждение:     #fbbf24   (жёлтый — акценты в кейсах)
```

**Логика выбора:** Фиолетовый акцент — нетипичный для разработчиков (обычно синий или зелёный), что помогает запомниться. Ассоциируется с ИИ-тематикой (OpenAI, Anthropic используют похожие оттенки).

## 8. Анимации и интерактив

### Принципы

1. **Не больше двух "сложных" анимаций** одновременно в viewport
2. **Все анимации уважают** `prefers-reduced-motion`
3. **Анимация усиливает смысл**, а не отвлекает от него
4. **Производительность прежде всего** — GPU-only свойства (transform, opacity)

### Анимации при загрузке (Hero)

```
Последовательность (stagger):
0.0s — фоновые частицы начинают рисоваться (fade in, 0.5s)
0.2s — тег над заголовком (fade + slide up 8px, 0.4s)
0.5s — H1 (посимвольное появление через Framer Motion, 0.8s)
1.0s — H2 (fade in + slide up, 0.4s)
1.2s — строка доверия (fade in, 0.3s)
1.4s — кнопки (fade in + scale 0.95→1, 0.3s)
1.6s — статистика (fade in + счётчики, 0.6s)
1.8s — фото (fade in + slide from right 16px, 0.5s)
```

### Scroll-анимации (GSAP ScrollTrigger)

```
Trust Signals:   счётчики от 0 при попадании в viewport
Services:        карточки появляются поочерёдно (stagger 0.15s)
Portfolio:       карточки снизу-вверх (stagger 0.1s)
Process:         шаги появляются слева-направо + линия "рисуется"
Testimonials:    fade in при появлении в viewport
Pricing:         карточки снизу-вверх (stagger 0.2s)
```

### Hover-анимации

```
Карточки кейсов:
  — border-color: transparent → акцентный (0.2s)
  — box-shadow: добавляется glow (0.2s)
  — изображение: scale(1.03) (0.3s)
  — кнопка внутри: opacity 0→1, translateY 4→0 (0.2s)

Карточки услуг:
  — background: слегка светлее (0.2s)
  — иконка: scale(1.1) + rotate(5deg) (0.3s)

Кнопки:
  — background: светлее на 10% (0.15s)
  — box-shadow: glow усиливается (0.15s)
  — transform: translateY(-1px) (0.15s)

Ссылки в навигации:
  — color: текст → акцентный (0.15s)
  — подчёркивание анимированное (width 0→100%, 0.2s)
```

### Фоновые частицы (Hero)

```javascript
// Параметры tsParticles / Canvas
particles: {
  count: 60,           // не больше — производительность
  color: '#a78bfa',    // акцентный фиолетовый

  size: 1–3px,
  speed: 0.3,          // очень медленно
  links: {
    enable: true,      // нейронные связи
    distance: 150,
    opacity: 0.08
  },
  interactivity: {
```
