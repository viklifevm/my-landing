## Фаза 2a. Мобильная адаптация (CSS-pass, 1 коммит «Mobile adaptation pass»)

> Сайт задеплоен на https://muzychenko-ai.vercel.app/. Обнаружены пробелы в трёх существующих брейкпоинтах (1024 / 768 / 480 px). Правки только в CSS + минимальные HTML-доработки для hero-фото.

### 🔴 Главное

- [ ] **Hero-фото видно на мобильных.** Сейчас `.hero-photo { display: none }` на ≤1024px — hero без визуала автора. Решение: вернуть фото компактно сверху над текстом (≤1024px → grid в 1 колонку, фото первой строкой, шириной ~140–180px, по центру).
- [ ] **Canvas-частицы выключить на ≤768px** — `@media (max-width:768px) { #particles { display:none; } }`. Совпадает с пунктом из Фазы 2 — делаем здесь.
- [ ] **Floating `tg-float`** — порог появления при скролле >50%. Совпадает с Фазой 2.
- [ ] **iOS auto-zoom на формах** — задать `input, textarea { font-size: 16px }` в мобильном брейкпоинте (Safari зумит, если меньше 16px).

### 🟡 Узкие места

- [ ] **Минимальная высота кликабельных элементов** — `.btn, .nav-link, a.social-link, .mobile-link { min-height: 44px }` (требование CLAUDE.md, сейчас не задано явно).
- [ ] **Шапка на 320–360px** — проверить, что `nav-logo` («Фаундер.Код») + бургер не вылезают; при необходимости уменьшить логотип до 1.1rem.
- [ ] **Вертикальный ритм** — у секций на мобильных слишком большой `padding`. Сократить на ≤768px (с 100px → ~64px) и на ≤480px (→ ~48px).
- [ ] **`<br>` в H1** — на ≤480px убрать жёсткий перенос, чтобы не было обрезки строк («Ваш бизнес<br>теряет деньги»).
- [ ] **Hero-stats** — на 320px цифры `2rem` могут быть мелковаты, проверить и при необходимости поднять до `1.75rem` с уменьшением gap.
- [ ] **Pricing-card на ≤480px** — длинные пункты списка проверить на overflow; уменьшить горизонтальный padding карточки.
- [ ] **Losses-cta, audit-card, cta-inner** — кнопки `width: 100%` на мобильных, чтобы не вылезали за края.
- [ ] **Footer на 768px** — проверить порядок колонок и отступы; иконки соцсетей выровнять по центру.
- [ ] **Кейсы / cases-grid** — на 480px проверить, что заголовки не обрезаются, тэги категорий не уходят в две строки уродливо.
- [ ] **`overflow-x: hidden` на body** — добавить как страховку от случайного горизонтального скролла.

## Фаза 3. Диагностический квиз = виджет на лендинге (4–6 часов)

> Объединяем «квиз» и «нейропродавец-виджет» в **один блок** на странице.
> Виджет в нынешнем виде (`neuro-seller.html`) **нерабочий** — fetch к Anthropic из браузера падает на CORS, ключ нельзя класть в JS, лиды пишутся только в console.log. Поэтому делаем как **квиз без AI**, ведущий в Telegram-бота.

### Архитектура
- 5 экранов с кнопками-ответами (без свободного ввода)
- В конце: персонализированный вывод (по матрице ответов) + кнопка «Записаться на диагностику»
- Кнопка ведёт в `t.me/founder_code_bot?start=<payload>`, где payload = код профиля квиза (3-4 буквы, например `prod-50-sales-no-many` сжатый до 4 символов)
- Бот при `/start <payload>` декодирует профиль и приветствует со ссылкой на боли клиента

### Вопросы (ответы — кнопки)
1. **Сфера бизнеса:** Производство · Услуги · Торговля · IT/digital
2. **Размер команды:** До 10 · 10–50 · 50+
3. **Где больше всего ручной работы:** Продажи · Отчётность · Производство · Коммуникации
4. **Есть ли CRM/ERP:** Нет · Частично · Да
5. **Часов рутины в неделю:** До 5 · 5–20 · 20+

### Размещение
- В `index.html` между секциями «Услуги» и «Цены»
- Стиль из существующих CSS-переменных, без новых шрифтов и библиотек
- Файл `neuro-seller.html` после интеграции **удаляем** — он будет дублирующим артефактом

### Что НЕ делаем сейчас
- AI-чат в виджете (требует серверного прокси — отдельная фаза в будущем)
- Свободный текстовый ввод
- Сохранение лидов из квиза в БД (всё уходит в Telegram-бота)

## Фаза 4. Деплой Telegram-бота (≈ 1 час)

> Пока бот не запущен где-то постоянно — форма на сайте бьёт в «никуда».

- [ ] Добавить `ANTHROPIC_API_KEY` в `.env` (получить на https://console.anthropic.com)
- [ ] Выбрать хостинг (рекомендация: Railway или Render, free tier хватает)
- [ ] Поднять воркер из `bot.py` (не web-service, а worker — polling)
- [ ] Прописать те же переменные окружения в панели хостинга
- [ ] Smoke-тест: написать `/start` боту, дойти до `[LEAD_READY]`, убедиться что уведомление прилетает на `OWNER_CHAT_ID = 302440535`
