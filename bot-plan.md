# Telegram-бот для лендинга Фаундер.Код

## Задача
AI-нейропродавец, который квалифицирует лидов с сайта и присылает резюме разговора владельцу.

## Как работает
1. На сайте клиент заполняет форму («Разобрать мой бизнес»): имя, контакт, описание задачи.
2. JS-обработчик кодирует имя+контакт в `start_param` (base64url, до 64 символов) и открывает в новой вкладке `https://t.me/founder_code_bot?start=<payload>`.
3. Бот при `/start <payload>` декодирует имя+контакт, приветствует клиента по имени и начинает диалог.
4. После 3–4 квалифицирующих вопросов бот предлагает диагностику. Когда лид готов — добавляет тег `[LEAD_READY]` в ответе.
5. По тегу бот вызывает Anthropic для краткого резюме и отправляет владельцу (`OWNER_CHAT_ID`) карточку лида с данными формы и резюме разговора.

## Стек
- Python 3.10+
- `python-telegram-bot` (polling)
- `anthropic` (Claude — диалог + резюме)
- Без БД, состояние в памяти процесса

## Зависимости и запуск
```bash
pip install -r requirements.txt
cp .env.example .env   # затем заполнить значения
python bot.py
```

## Переменные окружения (`.env`)
- `TELEGRAM_BOT_TOKEN` — токен от @BotFather (бот: `@founder_code_bot`)
- `ANTHROPIC_API_KEY` — ключ Anthropic
- `OWNER_CHAT_ID` — chat_id владельца, куда падают лиды (сейчас `302440535`)

⚠️ **Файл `.env` в git не коммитится** (см. `.gitignore`). Для деплоя задайте те же переменные в панели хостинга (Railway/Render/VPS).

## Деплой
Бот должен быть постоянно запущен. Варианты:
- VPS + systemd / pm2
- Railway / Render (worker, не web)
- Локально на сервере владельца

## Поля формы → бот
| Поле | Как доставляется |
|---|---|
| Имя | через `start_param` (base64url) |
| Email/Telegram | через `start_param` (base64url) |
| Описание задачи | сохраняется в `localStorage` сайта; бот спрашивает заново в первом вопросе |

Лимит `start_param` Telegram — 64 символа, поэтому описание задачи не передаётся напрямую. AI-бот всё равно задаёт уточняющие вопросы.
