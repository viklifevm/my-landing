## Правила редактирования index.html

### Что нельзя трогать без явной просьбы
- Цветовую схему (CSS-переменные в `:root`)
- Порядок секций
- Контактные данные (email, телеграм, макс)
- Имена и бренд

### CSS
- Все стили — внутри `<style>` в `<head>`, один блок
- Не добавлять `!important` без нужды (уже есть только там, где необходимо перебить наследование)
- Анимации через CSS `@keyframes`, не через JS-библиотеки
- Новые правила добавлять в конец соответствующего смыслового блока, не в хаотичном порядке

### JS
- Весь JS — в одном `<script>` перед `</body>`
- Не использовать `document.write`, `eval`, `innerHTML` для пользовательского ввода
- Новые обработчики добавлять после существующих, в конец скрипта

### Мобильная совместимость
- Брейкпоинты: 1024px (планшет), 768px (мобильный), 480px (маленький экран)
- Проверять, что новые элементы корректно отображаются на 320px–480px
- Кнопки и ссылки — минимум 44px по высоте для удобного тапа
